## 1. FaultRay の概要

FaultRay は「ゼロリスクインフラカオスシミュレーション」プラットフォームです。ユーザーが YAML・Terraform・クラウドプロバイダー連携でインフラトポロジーを定義し、実インフラには一切触れることなく数学的シミュレーションによって障害シナリオを解析します。レジリエンススコア・可用性上限・カスケード障害リスクを定量化し、CLI・REST API・Web ダッシュボードで結果を提供します。現在はオープンソース（自己ホスト）版とクラウド SaaS 版の両方を提供しており、特に金融機関の ICT リスク管理および EU DORA（デジタル・オペレーショナル・レジリエンス法）準拠ユースケースに注力しています。

## 2. ビジネスモデル

| 項目 | 内容 |
|------|------|
| **ライセンス** | BSL 1.1（Business Source License 1.1）— 一定期間後に OSS 転換予定 |
| **SaaS 料金体系** | Free / Pro / Enterprise の 3 ティア制（月次または年次課金、USD 建て） |
| **Free ティア** | セルフホスト相当の制限付き無償枠（シミュレーション 500 件・コンポーネント 10 個まで） |
| **Pro ティア** | SOC 2 / ISO 27001 コンプライアンスレポート付き、メール SLA 48h |
| **Enterprise ティア** | DORA・HIPAA・PCI DSS レポート対応、SSO/SAML、マルチテナント、専任サポート（SLA 4h） |

## 3. ターゲット市場

- **金融機関**（銀行・保険・証券など）: EU DORA（Regulation 2022/2554）第 6 条（ICT リスク管理）・第 28 条（ICT サードパーティリスク）準拠を求める事業者
- **SRE / DevOps チーム**: インフラ信頼性向上を目的とする技術チーム（エンタープライズ IT・スタートアップ問わず）
- **クラウドネイティブ企業**: Terraform・Kubernetes・Prometheus 環境を持ち、障害対策を自動化したい企業

## 5. レビュー依頼事項

以下の点について、法的リスクの指摘・改善提案をお願いします。

1. **利用規約（Terms of Service）のリスク箇所の指摘**
   - 現行草稿における法的に問題となりうる条項の洗い出し
   - 消費者契約法・特定商取引法との整合性確認（個人ユーザーが利用する場合）
   - 「シミュレーション結果はあくまで数学的モデルである」旨の免責文言の有効性

2. **Privacy Policy の GDPR / APPI 両対応の確認**
   - GDPR（EU Regulation 2016/679）の要求事項（同意取得・処理根拠の明記・データ主体の権利等）との整合性
   - 個人情報保護法（令和 2 年改正 APPI）との整合性（第三者提供の記録義務、越境移転規制等）
   - DORA 第 28 条のサードパーティリスク開示要件との整合性

3. **BSL 1.1 ライセンスの日本法での有効性**
   - Business Source License 1.1 が日本の著作権法・契約法のもとで有効に機能するかの確認
   - OSS 版（MIT ライセンス）との切り分けが適切かどうかの確認
   - BSL の「Change Date」以降の自動 OSS 転換条項に関するリスク

4. **免責条項の十分性（シミュレーション結果に基づく判断への責任制限）**
   - Section 6（Disclaimers）の免責・責任制限条項が日本法上有効かどうか（消費者契約法 8 条等との関係）
   - 金融機関ユーザーが FaultRay のシミュレーション結果を根拠に経営判断を行った場合の賠償リスク
   - 「AS IS」条項の日本語版での表現・有効性

5. **SaaS 利用規約として不足している条項の指摘**
   - 特定商取引法に基づく表記の要否
   - SLA（稼働率保証）に関するクレジット規定の法的有効性
   - 転売・再配布禁止条項の範囲と抜け穴
   - 未成年者・法人代理人の契約締結に関する条項の欠落有無
   - 反社会的勢力排除条項の要否
   - 準拠法・管轄の選択が外国ユーザーに対して有効かどうか
